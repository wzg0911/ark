问题摘要

🟠

RunnableWithFallbacks.stream()/astream() 用裸 next(stream) 偷看第一个 chunk 来判断主 runnable 是否"成功"。 但空流是完全合法的终态（LLM 返回空内容、过滤步骤、内容审核拦截）， 而 StopIteration 恰好是 Exception 子类， 被默认的 exceptions_to_handle=(Exception,) 当成真实故障捕获： 配了 fallback → 备胎输出无声替换合法的空输出； 没配 fallback → PEP 479 把 StopIteration 转成一个莫名其妙的 RuntimeError: generator raised StopIteration，彻底掩盖真实原因。

中高 · 静默替换 合法输出被当成故障 报错信息误导排障

影响范围

所有使用 with_fallbacks + 流式的链路（langchain-core）

根因定位

🧬 现象链

主 runnable 合法地产出 0 个 chunk（如审核拦截返回空）→ next(stream) 抛 StopIteration → 落入 except self.exceptions_to_handle（默认捕 Exception）→ 框架认定"主 runnable 挂了" → fallback 启动，产出替换内容 → 下游拿到的是备胎答案，且没有任何信号表明发生过替换。 没配 fallback 时，重新抛出的 StopIteration 在生成器内被 PEP 479 转成 RuntimeError: generator raised StopIteration—— 一条与业务毫无关系的报错。

⚙️ 根因

langchain_core/runnables/fallbacks.py 用"有没有第一个 chunk"这一存在性启发式来判断成功/失败， 把两个正交的问题（"流是否为空"与"流是否抛错"）折叠进同一个 except 分支。 StopIteration 作为控制流信号混入了异常处理通道—— 典型的"控制流信号被当成故障信号"缺陷。 修复只需 sentinel 窥视：next(stream, sentinel)。
💡 本质：fallback 机制的价值在于"故障时兜底"， 但"何为故障"的判定错了，兜底就变成了篡改。 比崩溃更糟的是静默替换：合规场景下"没有条款命中排除列表"的空结果被换成 fallback 内容， 下游没有任何信号。这正是 ARK OutputValidator（空输出是否合法终态可显式声明）+ OTel Bridge（fallback 触发必须留痕）的守备范围。

ARK 一键修复

✅ 方案A（推荐）：OutputValidator 让"空"成为显式契约，而不是模糊地带

from ark import OutputValidator

# 不变式：显式声明本节点空输出是否合法终态
schema = {
    "type": "object",
    "fields": {
        "chunks":  {"type": "number", "min": 0},          # 空流合法
        "source":  {"type": "string", "enum": ["primary"]}, # 输出必须来自主链路
    },
    "required": ["chunks", "source"],
}
validator = OutputValidator(schema)

result = collect(chain.stream(task))
validator.validate({"chunks": len(result), "source": trace_source(result)})
# 备胎无声上位 → source != "primary" → 立即 ValidationError，替换 1 秒暴露

💡 把"输出从哪来"从隐式假设升级成受检不变式——fallback 可以兜底，但不能不留痕迹地兜底。

✅ 方案B：OTel Bridge 给每次 fallback 触发留痕

# ARK 事件流：fallback 激活必产生可审计事件
# ark.fallback.activated {primary: "...", reason: "...", trace_id: "..."}
# 进入 Langfuse / Jaeger，"备胎替换了多少次主输出"从不可知变成一张报表

💡 本 issue 最痛的不是替换本身，而是零信号——留痕后即使上游未修复，损失也可量化、可回放。

✅ 方案C：上游修复（sentinel 窥视，社区已收敛的共识方案）

_MISSING = object()   # 模块级 sentinel

chunk = context.run(next, stream, _MISSING)
if chunk is _MISSING:
    return            # 空流 = 成功且为空，直接正常结束，不触发 fallback
