### 5.1 ⚠️ 发现次要问题：GitHub Action Publish to PyPI 失败
- **症状**：CI workflow `Publish to PyPI` → conclusion: failure
- **失败步骤**：`Run twine upload dist/* -u __token__ -p `（注意 `-p` 后是空的）
- **根因**：GitHub Repo 没有配置 `PYPI_TOKEN` secret
- **影响**：
  - PyPI 0.4.1 已存在（本地手动上传），✅ 用户无影响
  - 但 CI 自动发布路径断裂，**未来若想"打 tag 即发布"会失败**
- **修复方案**（需主人决策 + 一次性手动配置）：
  1. 去 https://pypi.org/account/ 申请 API token（限定 ark-trust 项目）
  2. 去 https://github.com/wzg0911/ark/settings/secrets/actions 添加 `PYPI_TOKEN`
  3. 重新打 tag 即可走通
- **未自动修复原因**：secret 涉及 PyPI 账号 + GitHub 仓库 settings，🟡 黄区操作，需主人授权
