### 8 种事件类型（OTel 语义约定扩展）
- `ark.idempotency.hit` / `ark.idempotency.miss`
- `ark.circuit.open` / `ark.circuit.close` / `ark.circuit.half_open`
- `ark.validation.fail` / `ark.validation.pass`
- `ark.guardian.intercept`

### 关键设计决策
1. **零依赖可选：** 纯标准库实现 OTLP/JSON，未设 `ARK_OTEL_ENDPOINT` 自动禁用，不污染 base 安装
2. **批量缓冲：** 100条/批 或 5秒刷新，降低 Collector 压力
3. **线程安全：** 缓冲用 RLock，Agent 并发场景下不丢事件
4. **优雅降级：** HTTP 失败计数但不抛异常（可靠性组件自身不能成为故障源）
5. **全局单例：** `get_otel_exporter()` 默认行为，Agent 框架集成时一行启用

### 使用示例
```python
import os
os.environ["ARK_OTEL_ENDPOINT"] = "http://otel-collector:4318/v1/events"

from ark import get_otel_exporter, EventType
exporter = get_otel_exporter()

# 在 IdempotencyGuard / CircuitBreaker / Validator 内自动 emit
exporter.emit(
    EventType.IDEMPOTENCY_HIT,
    tool_name="send_email",
    trace_id="abc",
    span_id="def",
    attributes={"saved_ms": 250.5}
)
```
