Q：ARK 凭什么敢说"永不崩溃"？

三层守护不是口号，是三道独立护栏：幂等守护拦截重复工具调用（同一动作只执行一次，杜绝重复扣费）；熔断在模型/服务故障时自动降级或切换，Agent 不卡死；输出校验拦截幻觉与格式错误，不让脏数据往下流。三层由 248 个测试守护，并在我们自己的生产环境连续运行 108 天验证。

Q：支持哪些语言和框架？

3语言+主流框架Python / TypeScript / Go 三套 SDK 功能全对齐；自动识别 LangChain、CrewAI、AutoGen/AG2、OpenAI SDK，零配置接入。自定义 Python Agent 只需一行 @guard.wrap 装饰器即可上护栏。

Q：我的数据和代码安全吗？

本地优先·MIT开源ARK 核心逻辑完全本地运行，不强制上云；整套工具包以 MIT 协议开源，代码可审计。OTel 链路追踪为可选模块——不接观测栈就零开销，接了才上报。

Q：ARK 和 LLM 网关 / 可观测平台有什么区别？

专注可靠性ARK 不是 LLM 网关，不替换你现有的模型调用，也不做统一路由。它是在你已有 Agent 之上加的一层可靠性护栏（幂等 / 熔断 / 输出校验 / 链路追踪），3 行接入、不改架构。可观测平台照常用，ARK 负责"别让 Agent 干出蠢事"。
